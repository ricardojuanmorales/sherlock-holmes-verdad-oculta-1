## 1. Decisiones oficiales

### 1.1 Formato base oficial
**Markdown (MD)** es el formato base oficial del proyecto para originales, plantillas, documentación maestra y entregables textuales.

### 1.2 Segundo Cerebro Oficial del Proyecto
El **Segundo Cerebro Oficial del Proyecto** es **Obsidian**.

### 1.3 Regla de interoperabilidad documental
Toda documentación crítica debe poder circular entre:

| Herramienta / Plataforma | Rol en el ecosistema |
|---|---|
| ChatGPT | Asistente de IA principal del sistema |
| LocalHost | Prueba de MVP y prototipos |
| UniqueUser | Entorno de usuario individual |
| VS Code | Editor de código y documentación |
| GitHub | Repositorio versionado oficial |
| GitHub Copilot | Asistente de programación autorizado |
| Obsidian | Segundo Cerebro Oficial del Proyecto |
| NotebookLM | Investigación y síntesis documental |
| Claude | Asistente de IA de soporte |
| Perplexity | Investigación con fuentes verificadas |
| ChatGPT Proyectos | Organización de sesiones de trabajo |

**Criterio:** sin pérdida semántica relevante y con trazabilidad mínima garantizada.

## 2. Términos normalizados

La siguiente tabla registra los cambios terminológicos aplicados a todos los documentos del proyecto a partir de la normalización V1.1.

| Término anterior (antes) | Término normalizado (ahora) |
|---|---|
| documentación espejo en Obsidian | registro estructurado en el Segundo Cerebro Oficial del Proyecto: Obsidian |
| base documental en Obsidian | Segundo Cerebro Oficial del Proyecto: Obsidian |
| sincronía con Obsidian | integración documental con el Segundo Cerebro Oficial del Proyecto: Obsidian |
| base Obsidian | Segundo Cerebro Oficial del Proyecto en Obsidian |
| formatos espejo: MD / JSON / HTML | formatos interoperables: Markdown (MD) / JSON / HTML |
| ruta_obsidian | ruta_segundo_cerebro_obsidian |
| Ruta espejo en Obsidian u otro sistema | Ruta en el Segundo Cerebro Oficial del Proyecto: Obsidian |

### Impacto de la normalización
- Eliminación de ambigüedad en referencias al sistema documental
- Uniformidad de metadatos en todas las plantillas YAML
- Consistencia entre expedientes, fichas de evidencia y bitácoras
- Alineación del vocabulario con el Mega_Glosario_Único

## 5. Regla operativa

A partir de esta normalización, el sistema opera bajo las siguientes reglas permanentes:

1. **Los originales se trabajan en Markdown (MD).** Ningún documento maestro vive solo en una plataforma de terceros.

2. **El sistema documental se articula con el Segundo Cerebro Oficial del Proyecto: Obsidian.** Todo metadato que referencie Obsidian usa el término normalizado completo.

3. **La interoperabilidad entre texto, código, evidencia y publicación es un criterio permanente de diseño.** No se añaden dependencias de plataforma cerrada sin evaluación explícita.

4. **Cuando un término del glosario cambie, se versionan todos los documentos afectados.** Los cambios terminológicos no son implícitos; se registran en este documento.

5. **La consistencia terminológica entre el Mega_Glosario_Único, las plantillas y la documentación de base es obligatoria.** El glosario es la fuente de verdad terminológica del proyecto.
